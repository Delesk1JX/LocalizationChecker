# LocalizationChecker 🔍

Программа для проверки русской локализации в модах Minecraft.  
**Работает напрямую с .jar файлами** — не нужно ничего распаковывать! Автоматически сканирует моды, проверяет наличие `ru_ru.json` внутри архивов и сравнивает ключи с `en_us.json`.

##  Возможности

- ✅ **Проверка наличия** русской локализации (`ru_ru.json`) прямо в .jar файлах
- 📊 **Определение полноты перевода**:
  - [100%] **Полный перевод** — 100% совпадение ключей с английским файлом (все ключи из en_us есть в ru_ru)
  - [Частично] **Неполный перевод** — есть русский файл, но не все ключи из английского присутствуют
  - [Нет] **Отсутствует** — нет файла `ru_ru.json`
- 💾 **Экспорт результатов** в JSON-файл с полной информацией
- 🔍 **Проверка по содержанию** — программа сравнивает именно ключи, а не количество строк
- 📦 **Сохранение лишних ключей** — если в ru_ru.json есть ключи, которых нет в en_us.json, они сохраняются в отчете (для обратной совместимости со старыми версиями)
- 🖥️ **Графический интерфейс** — удобный GUI с вкладками, прогресс-баром и детальным просмотром
- ⚡ **Многопоточная обработка** — быстрая проверка даже 400+ модов

## 🛠️ Использование

### Графический интерфейс (рекомендуется)

Просто запустите программу:

```bash
# Из исходного кода
python check_localization.py

# Или готовый исполняемый файл
./dist/LocalizationChecker
```

В GUI вы сможете:
1. Нажать "📁 Выбрать папку с модами" и указать директорию с .jar файлами
2. Нажать "▶️ Проверить" для начала сканирования
3. Просмотреть результаты по вкладкам ([100%] Полный, [Частично] Неполный, [Нет] Отсутствует)
4. Двойной клик на моде покажет детали (списки недостающих и лишних ключей)
5. Нажать "💾 Экспорт в JSON" для сохранения отчета

### Консольная версия

```bash
# Проверка текущей директории
python check_localization.py .

# Проверка конкретной папки с .jar файлами
python check_localization.py C:\Games\Minecraft\mods

# Сохранить в другой файл
python check_localization.py -o results.json

# Запустить GUI явно
python check_localization.py --gui
```

## 📤 Формат выходного файла

Результаты сохраняются в JSON-файл (по умолчанию `localization_results.json`):

```json
{
  "scan_directory": "/home/user/mods",
  "total_mods": 150,
  "summary": {
    "full_translation": 45,
    "partial_translation": 30,
    "missing_translation": 75
  },
  "mods": {
    "full": [
      {
        "mod_name": "mod_with_full_translation.jar",
        "status": "full",
        "ru_keys": 120,
        "en_keys": 120,
        "percentage": 100.0,
        "missing_keys": [],
        "extra_keys": []
      }
    ],
    "partial": [
      {
        "mod_name": "mod_partial.jar",
        "status": "partial",
        "ru_keys": 80,
        "en_keys": 100,
        "percentage": 80.0,
        "missing_keys": [
          "item.mod.key1",
          "block.mod.key2"
        ],
        "extra_keys": [
          "old.compat.key"
        ]
      }
    ],
    "missing": [
      {
        "mod_name": "mod_no_russian.jar",
        "status": "missing",
        "ru_keys": 0,
        "en_keys": 150,
        "percentage": 0.0,
        "missing_keys": [],
        "extra_keys": [],
        "error": null
      }
    ]
  }
}
```

### Поля в отчете:

- `mod_name` — имя .jar файла
- `status` — статус перевода (full/partial/missing)
- `ru_keys` — количество ключей в русском файле
- `en_keys` — количество ключей в английском файле
- `percentage` — процент переведенных ключей (отношение ключей из en_us, которые есть в ru_ru)
- `missing_keys` — список ключей из en_us, которых **нет** в ru_ru (нужно перевести)
- `extra_keys` — список ключей из ru_ru, которых **нет** в en_us (сохранены для совместимости)
- `error` — сообщение об ошибке, если возникла проблема при чтении файла

## 💡 Советы по использованию

1. **Ничего не нужно распаковывать** — программа работает напрямую с .jar файлами
2. **Лишние ключи сохраняются** — если в ru_ru.json есть ключи от старых версий мода, они не считаются ошибкой, а сохраняются в поле `extra_keys` для справки
3. **Используйте GUI для удобства** — графический интерфейс позволяет быстро просматривать детали по каждому моду
4. **Анализируйте missing_keys** — в JSON-отчете есть точный список ключей, которые нужно перевести
5. **Сортируйте моды** — начните с модов, у которых высокий процент перевода (80-99%), их легче всего завершить

## ⚠️ Важные замечания

- **Полный перевод = строго 100%** — даже один отсутствующий ключ делает перевод неполным
- Программа сравнивает **ключи**, а не значения — важно наличие ключа, а не качество перевода
- **Лишние ключи в ru_ru** — если в русском файле есть ключи, которых нет в английском, они сохраняются в отчете (поле `extra_keys`), так как могут использоваться для обратной совместимости со старыми версиями мода
- Работает только с .jar файлами, содержащими структуру `assets/<mod_id>/lang/`
- Поддерживаются оба формата путей в архиве: `/lang/` и `\lang\`
